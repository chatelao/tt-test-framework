# Spongent-88 Hash Accelerator

**Source:** [https://github.com/imix/ttihp-spongent88](https://github.com/imix/ttihp-spongent88)

**TinyTapeout Project Page:** [https://app.tinytapeout.com/projects/4073](https://app.tinytapeout.com/projects/4073)

## Overview

Hardware accelerator for the **Spongent-88/80/8** lightweight hash function, specifically optimized for **Winternitz One-Time Signatures (W-OTS)**. This project implements a high-performance, gate-efficient silicon core for the Tiny Tapeout IHP shuttle.

## How it Works

### Spongent-88/80/8 Algorithm

Spongent is a sponge-based hash function designed for extremely constrained hardware environments. The 88/80/8 variant implemented here features:
- **88-bit internal state**: 8-bit rate ($r$) and 80-bit capacity ($c$).
- **45 permutation rounds**: Each absorption step triggers a full permutation.
- **2-Round Unrolling**: The permutation engine computes two rounds per clock cycle, completing the 45-round hash in just 23 cycles.

The round function consists of:
1. **Counter Injection**: A 6-bit LFSR round constant is XORed into the state.
2. **S-Box Layer**: 22 parallel 4-bit Spongent S-boxes provide non-linear confusion.
3. **pLayer**: A bit-shuffle permutation ($P(j) = (j \times 22) \pmod{87}$) provides global diffusion.

### Sponge Construction

Host software absorbs message bytes one at a time. Each byte is XORed into the rate portion (`state[7:0]`), followed by a permutation. After all bytes (plus padding) are absorbed, the 88-bit state forms the digest. The chip supports the `pad10*1` rule, which for byte-aligned messages results in a final `0x81` byte.

## Register Interface

The chip is controlled through a byte-serial interface using the TinyTapeout I/O pins.

### Pin Mapping

| Signal | Direction | Description |
|---|---|---|
| `ui_in[7:0]` | input | Data byte to write |
| `uo_out[7:0]` | output | Current digest byte (LSB-first) |
| `uio_in[2:0]` | input | Register Address |
| `uio_in[3]` | input | Write Strobe (rising-edge) |
| `uio_in[4]` | input | Read Strobe (advances output) |
| `uio_out[0]` | output | **Busy**: High while permutation |
| `uio_out[1]` | output | **Out Valid**: High after squeeze |

### Register Map

| Addr | Name | Direction | Action |
|---|---|---|---|
| 0 | **CMD** | Write | `0`=Reset, `1`=Squeeze, `2`=Hash (Auto-Pad) |
| 1 | **ABSORB**| Write | XORs `ui_in` into state and starts permutation |
| 2 | **RD_ADV**| Strobe | Advances output shift register to next byte |

**Timing**: One absorption call takes 25 clock cycles (1 load + 23 permutation + 1 capture).

## W-OTS Use Case

Winternitz One-Time Signatures (W-OTS) are a post-quantum secure signature scheme. With parameter $w=16$, it uses 25 hash chains of depth 15. This accelerator speeds up the 375 permutation calls required per signature. At 50 MHz, a full signature's hashing is completed in approximately **190 µs**.
